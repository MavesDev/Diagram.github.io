Structure Diagram

Diagram struktur adalah jenis diagram dalam Unified Modeling Language (UML) yang digunakan untuk menggambarkan struktur atau komponen dari suatu sistem atau bagian dari sistem tersebut.

Diagram Class

Class diagram adalah salah satu jenis diagram dalam UML yang digunakan untuk menggambarkan struktur sistem dengan menunjukan kelas – kelas yang ada didalam sistem, atribut, metode, dan hubungan antar objek.

  • Menggambarkan struktur dari sebuah sistem
  • Representasi struktur statik
  • Pemahaman konsep objek
  • Menunjukan hubungan antar kelas

Class

Entitas dasar dalam diagram kelas, mewakili jenis atau tipe objek yang akan dibuat didalam sebuah sistem.

Attribute

Merupakan variabel atau data yang dimiliki oleh sebuah kelas

Method

Merupakan fungsi atau operasi yang dapat dilakukan oleh sebuah kelas

Connection

Hubungan menggambarkan cara kelas – kelas berinteraksi satu sama lain dalam sebuah sistem

Diagram Object

Object diagram merupakan jenis diagram dalam permodelan berorientasi objek yang digunakan untuk menggambarkan instance dari kelas – kelas dalam suatu sistem. Diagram ini menunjukan hubungan antara objek - objek dan atribut atributnya.

  • Diagram ini digunakan biasanya dalam membuat prototype
  • Menunjukan hubungan antara objek – objek dan atributnya
  • Memberikan gambaran tentang sistem atau perangkat lunak serta relasi relasinya
  • Bisa digunakan untuk menguji keakuratan diagram class

Object

Objek merupakan instance dari sebuah kelas dalam sistem

Attribute

Merupakan variabel atau data yang dimiliki oleh sebuah objek

Link

Garis yang menghubungkan satu garis dengan garis yang lainnya.

Diagram Component

Diagram component menggambarkan kumpulan komponen dan hubungan antar komponen juga memperlihatkan sebuah organisasi serta keterkaitan dan ketergantungan diantara komponen pada sistem.

  • Sebagai source code perangkat lunak (software)
  • Komponen executeable yang dilepas dan digunakan user
  • Sebagai framework sistem

Package

Sebuah symbol yang digunakan untuk mewadahi komponen

Component

Merupakan symbol yang menggambarkan hardware atau objek pada sistem

Dependency

Simbol yang menggambarkan ketergantungan antar komponen, satu komponen dengan yang lainnya.

Link

Simbol yang digunakan untuk menunjukan relasi antar komponen

Diagram Deployment

Deployment diagram merupakan diagram yang menjelaskan secara detail bagaimana komponen disebar atau dideploy kedalam infrastruktur sebuah sistem.

  • Menunjuk struktur dari sistem run-time
  • Memberikan gambaran bagaimana hardware memiliki keterkaitan satu sama lain
  • Menunjukan hardware dan software yang diinstal

Package

Merupakan sebuah bungkusan dari satu atau lebih node

Node

Simbol yang biasanya mengacu pada hardware, software yang tidak dibuat sendiri.

Dependency

Kebergantungan antar node, arah panah mengarah pada node yang dipakai

Link

Relasi antar node

Diagram Package

Package diagram atau diagram paket merupakan salah satu jenis UML yang dipakai untuk mengelompokkan elemen-elemen seperti use case maupun class diagram.

  • memberikan overview sekumpulan kebutuhan dan desain arsitektur dari sebuah sistem yang memiliki hubungan logis dalam diagram modularnya (memecah sistem menjadi bagian kecil).

Package

Packages berisikan sekumpulan elemen UML yang saling memiliki hubungan logical

Import

Notasi import mengartikan sebuah package mengimport sebuah fungsi dari Package lain dan memberikan One Way Permission.

Access

Notasi acces mengartikan sebuah package membutuhkan fungsi dari package lain untuk menjalankan fungsi package itu sendiri.

Diagram Profile

Diagram profil adalah mekanisme ekstensibilitas yang memungkinkan untuk memperluas dan menyesuaikan UML dengan menambahkan blok penyusun baru, membuat properti baru, dan menentukan semantik baru agar bahasanya sesuai dengan domain masalah.

  • Profile diagram digunakan untuk memperluas atau menyesuaikan notasi dan semantik UML standar.

Stereotypes

Stereotip digunakan untuk memperkenalkan elemen penyusun baru yang sesuai dengan bahasa domain Anda dan terlihat primitif.

Tagged Values

Digunakan untuk memperluas properti UML sehingga bisa menambahkan informasi tambahan dalam spesifikasi elemen model.

Constraints

Untuk menentukan semantik atau kondisi yang harus selalu dijaga kebenarannya. Sehingga meungkinkan untuk memperluas semantik blok penyusun UML dengan menambahkan protokol baru.

Diagram Behaviour

Diagram perilaku adalah jenis diagram dalam Unified Modeling Language (UML) yang digunakan untuk menggambarkan perilaku dinamis dari sistem atau komponen dalam sistem. Diagram ini lebih berfokus pada bagaimana objek-objek dalam sistem berinteraksi dan berperilaku selama waktu tertentu.

Diagram UseCase

Use case diagram merupakan pemodelan fungsional untuk menggambarkan berhaviour atau kelakuan sebuah sistem yang akan dibangun. Use case diagram tidak menjelaskan secara rinci mengenai penggunaan use case dan hanya memberikan gambaran singkat antara use case, actor dan sistem

  • Memperlihatkan proses aktivitas secara urut dalam sistem
  • Menggambarkan proses bisnis serta aktivitas pada sebuah proses

Sistem

Sistem pada use case digambarkan dengan bentuk persegi Panjang yang membatasi use case dengan interaksi diluar sistem

Aktor

Mewakili peran orang, sistem lain, atau alat ketika berkomunikasi dengan use case

Use Case

Abstraksi dan interaksi antara sistem dan aktor

Association

Abtraksi dari penghubung antara aktor dan use case

Generalisasi

Spesialisasi aktor untuk dapat berpartisipasi dengan use case

Include

Menunjukan bahwa suatu usecase merupakan fungsionalitas dari use case lainnya

Extend

Menunjukan bahwa suatu usecase merupakan tambahan fungsional dari use case lainnya jika suatu kondisi terpenuhi

Diagram Activity

Diagram yang dapat memodelkan proses-proses yang terjadi pada sebuah sistem. Runtutan proses dari suatu sistem digambarkan secara vertical. Activity diagram merupakan pengembangan dari Use Case yang memiliki alur aktivitas.

  • Menjelaskan urutan aktivitas dalam suatu proses.
  • Di dalam dunia bisnis biasanya digunakan untuk modeling (memperlihatkan urutan proses bisnis).
  • Mudah dalam memahami proses yang ada dalam sistem secara keseluruhan.
  • Merupakan metode perancangan yang terstruktur, mirip dengan Flowchart maupun Data Flow Diagram (DFD).
  • Mengetahui aktivitas aktor/pengguna berdasarkan use case/diagram yang dibuat sebelumnya.

Start

Status awal pada sebuah diagram aktivitas

End

Status akhir pada seuah diagram aktivitas

Activity

Aktivitas yang dilakukan oleh sistem

Branch

Percabangan dimana ada pilihan aktivitas yang lebih dari satu

Join

Penggabungan dimana lebih dari satu aktivitas lalu digabung menjadi satu

Swimlane

Memisahkan organisasi bisnis yang bertanggung jawab terhadap aktivitas yang terjadi

Diagram Statechart

Statechart atau state diagram adalah sebuah diagram yang menggambarkan, memperlihatkan serta mendeskripsikan tentang perilaku sistem. Pada umumnya statechart diagram ini menggambarkan seluruh kondisi yang ada sebagai sebuah objek ataupun event

  • Memodelkan sistem reaktif, yaitu sebuah sistem yang merespon kejadian internal ataupun external.
  • Memodelkan sebuah aktivitas life time dari awal hingga akhir.
  • Memodelkan aspek dinamik suatu sistem.
  • Memodelkan masa pakai sistem reaktif
  • Menggambarkan objek yang beda ketika sistem sedang berjalan.

Transition

Merupakan gambar sebuah panah yang menunjukan jalur bagian berbeda dari state sebuah objek. Transisi tidak hanya mengarah ke state lain, akan tetapi dapat menunjuk balik ke dirinya sendiri.

Initial State

Sebuah panah yang menunjuk pada lingkaran

Final State

Notasi ini menunjukan keadaan akhir objek

Synchronization and Splitting of Control

Sebuah bar pendek dengan dua transisi entering merupakan sinkronisasi kontrol. Bar pertama sering disebut fork yang dimana transisi tunggal terbagi menjadi beberapa transisi. Bar kedua disebut join, dimana transisi bersamaan mengurangi kembali ke satu.

Diagram Sequence

Sequence Diagram adalah fokus waktu dan menunjukkan urutan interaksi secara visual dengan menggunakan sumbu vertikal diagram untuk mewakili waktu, pesan apa yang dikirim dan kapan.

  • Memodelkan interaksi tingkat tinggi antara objek aktif dalam suatu sistem
  • Modelkan interaksi antar instance objek dalam kolaborasi yang merealisasikan use case
  • Memodelkan interaksi antar objek dalam kolaborasi yang mewujudkan suatu operasi
  • Entah memodelkan interaksi umum (menunjukkan semua jalur yang mungkin melalui interaksi) atau contoh interaksi tertentu (menunjukkan hanya satu jalur melalui interaksi)

Actor

Jenis peran yang dimainkan oleh entitas yang berinteraksi dengan subjek (misalnya, dengan bertukar sinyal dan data)

Activation

Persegi panjang tipis pada garis hidup mewakili periode di mana suatu elemen melakukan suatu operasi. bagian atas dan bawah persegi panjang disejajarkan dengan waktu inisiasi dan waktu penyelesaian

Call Message

Pesan panggilan adalah jenis pesan yang mewakili pemanggilan pengoperasian jalur hidup target.

Return Message

Jenis pesan yang mewakili penyampaian informasi kembali ke pemanggil pesan sebelumnya yang berkorespondensi.

Self message

Jenis pesan yang mewakili pemanggilan pesan dari jalur hidup yang sama.

Recursive message

Jenis pesan yang mewakili pemanggilan pesan dari jalur hidup yang sama. Ini adalah titik target ke aktivasi di atas aktivasi tempat pesan itu dipanggil.

Create Message

Jenis pesan yang mewakili contoh garis hidup (target).

Destroy Message

Jenis pesan yang mewakili permintaan untuk menghancurkan siklus hidup dari garis hidup target.

Duration Message

Menunjukkan jarak antara dua waktu instan untuk pemanggilan pesan.

Note

Note memberikan kemampuan untuk melampirkan berbagai komentar ke elemen. Sebuah komentar tidak memiliki kekuatan semantik, namun mungkin berisi informasi yang berguna bagi pemodel.

Diagram Interaction

Diagram yang digunakan untuk menggambarkan bagaimana objek-objek dalam sistem berinteraksi satu sama lain dalam konteks kasus penggunaan tertentu.

  • Untuk menangkap perilaku dinamis suatu sistem.
  • Untuk menggambarkan aliran pesan dalam sistem.
  • Untuk menggambarkan organisasi struktural objek.
  • Untuk menggambarkan interaksi antar objek.

Diagram Timing

Timing Diagram adalah diagram yang dipakai untuk mengatasi tingkah laku obyek sepanjang jangka waktu tertentu.

  • Timing diagram UML berfungsi untuk memunculkan perubahan dalam keadaan / nilai dari satu atau lebih elemen dari waktu ke waktu.

Lifeline

Garis hidup adalah elemen utama dalam diagram waktu dan digunakan untuk mewakili objek atau entitas dalam sistem yang terlibat dalam interaksi.

Activation Bar

Bar aktivasi digunakan untuk menunjukkan periode waktu di mana objek atau entitas tertentu aktif atau sedang melakukan sesuatu.

Message

Pesan digunakan untuk menggambarkan komunikasi atau interaksi antara objek-objek dalam diagram waktu.

Self-Message

Pesan diri sendiri digunakan ketika objek mengirim pesan kepada dirinya sendiri.

Constraint

Kondisi digunakan untuk menunjukkan syarat atau kondisi yang harus dipenuhi sebelum pesan dapat dikirim atau sebelum objek dapat memasuki atau meninggalkan aktivitas tertentu.

Observation

Pengamatan digunakan untuk menunjukkan pemantauan atau pengawasan terhadap objek selama aktivitas berlangsung.

Time Constraint

Kondisi waktu digunakan untuk menunjukkan batasan waktu pada aktivitas atau pesan.

Duration Constraint

Kondisi durasi digunakan untuk menunjukkan durasi aktivitas atau pesan dalam satuan waktu tertentu.

Continuation

Kelanjutan digunakan untuk menggambarkan kelanjutan dari aktivitas atau pesan di tempat lain dalam diagram waktu.

Gate

Gerbang digunakan untuk menggambarkan titik masuk atau keluar dari aktivitas atau pesan.

Fragment

Fragment digunakan untuk menggambarkan bagian-bagian dari diagram waktu yang terkait dengan kondisi atau perulangan tertentu.

Diagram Communication

Communication Diagram adalah suatu diagram yang mengambarkan struktur interaksi yang terdapat disekitar objek (seperti halnya sequence diagram) dan hubungannya terhadap yang lainnya.

  • Model pesan melalui antar peran atau objek yang mengirimkan operasi penggunaan dan fungsionalitas kasus.
  • Mekanisme model dalam desain arsitektur system.
  • Tangkap interaksi yang mengarahkan pesan yang dilewati antar peran dan objek dalam skenario kolaborasi.
  • Model skenario alternatif dalam kasus pemakaian atau operasi yang melibatkan kolaborasi dari berbagai interaksi dan objek.
  • Dukung pengidentifikasian objek (sebab kelas), atributnya (parameter pesan), dan operasi (pesan) yang ikut serta dalam use case.

Object

Objek adalah instance dari sebuah kelas yang tersusun secara horizontal. Digambarkan sebagai sebuah kelas (kotak) dengan nama objek didalamnya yang diawali dengan sebuah titik koma (;).

Actor

Actor juga dapat berhubungan dengan objek , maka actor juga dapat diurutkan sebagai kolom. Simbolnya sama pada Actor Use Case Diagram.

Message

Message digambarkan dengan anak panah yang mengarah antar obyek dan diberi label urutan nomor yang mengindikasikan urutan komunikasi yang terjadi antar obyek, seperti yang sudah digambarkan dan dijelaskan di atas.

Diagram Interaction Overview

Interaction Overview Diagram merupakan sebuah diagram yang digunakan untuk melakukan visualisasi mengenai kerja sama yang terjadi antara sequence diagram dengan activity diagram.

  • Memberikan abstraksi visual yang memudahkan pemahaman sistem secara cepat tanpa perlu detail teknis yang mendalam.
  • Komunikasi dengan Pemangku Kepentingan:
  • Sebagai alat navigasi untuk pengembang dan membantu dalam memahami cara pengguna akan berinteraksi dengan sistem.

Decision

Merupakan sebuah unsur pada Interaction Overview Diagram yang bertugas untuk menunjukkan adanya titik perkembangan yang bersyarat. Jika kondisi benar, maka proses akan terus berjalan

Initial

Simbol yang berfungi untuk mendefinisikan awalan dari aliran pada saat sebuah kegiatan sedang dipanggil

Final

Simbol yang befungsi menunjukkan akhir dari sebuah aktivitas

Activity

Simbol yang berfungsi untuk memberikan petunjuk seputar aktivitas atau kegiatan yang sedang berlangsung

Control Flow

Sebuah konektor yang berfungsi untuk menghubungkan dua node di dalam diagram aktivitas. Simbol ini akan berperan sebagai jembatan dari aliran node aktivitas di mana nantinya aliran tersebut akan mengarah ke node target
